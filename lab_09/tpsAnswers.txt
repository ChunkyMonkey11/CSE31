TPS 1

1) Downloaded and used the MIPS reference sheet.

2) Loaded proc1.s and reviewed source + text segment mapping.

3) Verified source instructions map to basic (true) instructions and machine code in Text Segment.

4) A MIPS machine instruction is 32 bits.

5) There are 3 instruction formats:
- R-type (examples: add, slt)
- I-type (examples: addi, beq)
- J-type (examples: j, jal)

6) Line #7: addi $s0, $zero, 25
6a) Type: I-type. Fields: opcode, rs, rt, immediate.
6b)
- opcode(addi) = 0x08
- rs = $zero = 0x00
- rt = $s0 = 0x10
- immediate = 25 = 0x0019
6c)
- Binary: 001000 00000 10000 0000000000011001
- 32-bit binary: 00100000000100000000000000011001
- Hex: 0x20100019

7) Address 0x00400010 (from Text Segment)
7a)
- Machine code (hex): 0x0230402A
- Binary: 00000010001100000100000000101010
7b)
- Type: R-type (opcode bits [31:26] = 000000)
- Fields: opcode, rs, rt, rd, shamt, funct
7c)
- opcode = 0x00
- rs = 0x11
- rt = 0x10
- rd = 0x08
- shamt = 0x00
- funct = 0x2A
7d)
- funct 0x2A => slt
- Registers: rs=0x11->$s1, rt=0x10->$s0, rd=0x08->$t0
7e)
- Final instruction: slt $t0, $s1, $s0
- Yes, it matches Source column.

8) Line #17: bne $t0, $zero, LESS
8a) Format: I-type
8b)
- opcode(bne)=0x05
- rs=$t0=0x08
- rt=$zero=0x00
8c)
- Target label: LESS
- LESS address: 0x0040001C
8d)
- We do NOT place the absolute target address in immediate.
- Branch immediate is a signed PC-relative offset in words.
8e)
- immediate = (target - (PC+4))/4
- PC of bne = 0x00400014, so PC+4 = 0x00400018
- immediate = (0x0040001C - 0x00400018)/4 = 1
- immediate = 0x0001
8f)
- Binary: 000101 01000 00000 0000000000000001
- 32-bit binary: 00010101000000000000000000000001
- Hex: 0x15000001
- Matches code column.

9) Line #20: j GREQ
9a)
- Format: J-type
- Fields: opcode, target(26 bits)
9b)
- opcode(j)=0x02
9c)
- Jumps to label GREQ at address 0x00400030
9d)
- Address field size: 26 bits.
- Encode target as address >> 2 (drop low 2 bits, word alignment).
- target field = 0x00400030 >> 2 = 0x0010000C
- target field binary (26 bits): 00000100000000000000001100
- Reason: instructions are word-aligned, so low 2 bits are always 00.
9e)
- Binary: 000010 00000100000000000000001100
- 32-bit binary: 00001000000100000000000000001100
- Hex: 0x0810000C
- Matches code column.
