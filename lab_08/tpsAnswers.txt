TPS 2
1. For input `5`, `recursion.c` returns `29`. During execution it prints several lines of `Returning 1` and `Returning 3`, then final output is `29`.

2. `main` in `recursion.s` sets up stack space, prompts for input, reads integer, passes it in `$a0`, calls `recursion`, and prints returned value.

3. I inserted prompt print instructions so assembly behavior matches C:
   `Please enter an integer: `

4. Input is read using syscall 5. The value is moved to `$a0` because `$a0` is the argument register used by `jal recursion`.

5. Return value comes back in `$v0`. Print syscall also uses `$v0` for syscall code, so I first save return value to a temp register (ex: `$t0`), then set `$v0=1`, move `$a0=$t0`, and print.

6. Added print sequence after `jal recursion` and before program exit, so returned value is printed correctly.

7. In `recursion`, stack frame reserves 3 integers (12 bytes):
   - 0($sp): saved `$ra`
   - 4($sp): saved argument `m` (`$a0`)
   - 8($sp): saved first recursive return value
   First thing stored is `$ra`.

8. Base case `m == -1`: set return register `$v0 = 1`.

9. Base case `m == 0`: set return register `$v0 = 3`.

10. We save `$a0` before recursive calls because `$a0` is overwritten when preparing new arguments (`m-2`, `m-1`). We still need original `m` for the second call setup.

11. For first recursive call, update argument:
   `addi $a0, $a0, -2`

12. After first call, returned value is in `$v0`. If we call recursion again immediately, that value gets overwritten. So store first result to stack (ex: `sw $v0, 8($sp)`), restore original `m`, then prepare second call.

13. For second call, set argument to `m - 1`:
   `lw $a0, 4($sp)`
   `addi $a0, $a0, -1`

14. After second call, combine results:
   `lw $t0, 8($sp)`
   `add $v0, $t0, $v0`
   so function returns `recursion(m-2) + recursion(m-1)`.

15. Before `jr $ra`, restore saved return address:
   `lw $ra, 0($sp)`
   then pop stack frame and return.
